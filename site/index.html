What it catches

Seven finding kinds. Every one derivable from the diff.

No inference, no opinion. Stage 3 is pure set arithmetic: actual − authorized = findings. If it isn't in the diff, it isn't a finding.

HIGH
scope.env

An env var the prompt never named — process.env.STRIPE_SECRET, os.environ[...], a new .env line.

HIGH
scope.endpoint

An HTTP route or handler added without authorization — FastAPI/Flask decorators, Express/Next/Hono routes, route.ts files.

HIGH
scope.cron

A scheduled job that appeared from nowhere — cron.schedule, BackgroundScheduler, Vercel cron, @Cron.

HIGH
scope.listener

A runtime surface the prompt never opened — app.listen(), a WebSocket.Server, process.on, window.addEventListener.

MEDIUM
scope.dep

A package the prompt didn't ask for — a new line in package.json / composer.json, requirements.txt, go.mod, Cargo.toml, or Gemfile.

MEDIUM
scope.file

Edits to a file the prompt never touched on — the agent wandered outside the requested path.

LOW
scope.feature

A new top-level symbol or feature beyond what was asked — functions, classes, exports the prompt didn't entail — including infrastructure: terraform resource, kubernetes kind:, CloudFormation, and SQL CREATE/ALTER TABLE.

How it works

Three stages. One LLM call. Two are pure functions.

01
LLM · cheap & fast · temp 0

Extract scope

One small model call reads your prompt and returns the authorized scope as strict JSON. It deciphers typos ("setings" → "settings") but never invents scope — only what you literally asked for. The only stage that calls a model.

02
Deterministic · no LLM · <100ms

Extract actual

Regex + light parsing turns the unified diff into exactly what it adds — imports, deps, env vars, routes, cron jobs, symbols. Per-language. Zero API cost, fully auditable.

03
Pure set arithmetic · instant

Diff

For each category, subtract anything the prompt authorized. What's left is a finding, with severity assigned. scope_creep_score: HIGH if any high-severity finding, MEDIUM if only medium, else LOW.

Scope DSL — proactive mode

Agents can declare what they will do before writing code via claim_scope. Validation is then fully deterministic — confidence 1.0, zero API cost, Stage 1 skipped entirely. Parent→child claims can only narrow, never expand. This is the upgrade from reactive auditing to locked, declared intent.

Multi-agent coordination

When three agents share one repo, nothing collides.

A coordination layer any agent can read — it's just JSON files in .overreach/ committed to git. Sixteen MCP tools. Cross-vendor by default.

file claims

Agents claim files before working. Others see the claims and work elsewhere. Claims auto-expire (2h default) and can be extended.

conflict resolution

Contested files are blocked or escalated for human review. Every conflict gets an id and a record.

coordination ledger

Every agent's work is logged — files touched, scope-creep score, mode (dsl/inferred), confidence, timestamp. Read before you start.

traceability

Trace any file change back to the ticket that caused it. who_touched("src/auth.ts") → the full agent history.

delegation chains

Parent→child handoffs can only narrow scope, never expand. Contracts carry an optional TTL — expired contracts flag HIGH so stale agents stop committing.

cross-vendor init

One command writes CLAUDE.md, .cursorrules, codex.md, a pre-commit hook, and the coordination config. Every vendor onboarded at once.
